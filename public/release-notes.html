Release notes

Aithernet is in invitation-only early access. Signed releases are delivered through the authenticated portal to approved customers. No public download is offered on this site, and there is no public general-availability release yet.

This site hosts no binaries, checksums, signatures or keys. When you are approved, your release — Debian package, wheel/sdist, signed manifest, checksums and the verification key — appears on the portal's Downloads page with the exact, versioned commands.

How releases work

  • Each release is an immutable, signed bundle: a detached signature over a manifest, per-artifact SHA-256 checksums, and an explicit signing key identifier.
  • Customers verify the signature and checksums with the published verification key before installing.
  • The RF MCP component ships its GPL corresponding source alongside the binary release.
  • A superseded qualification release is only revoked after its replacement has passed.

What is being qualified

The current early-access build (1.0.0-beta.3) adds the account-bound fleet, tenant-isolated secure meshes, and the complete OTA link protocol on top of the operator-controlled SDR peer transport. It has completed and passed the following software qualification, automatically, on a clean source-hidden install:

  • production-signed release publication and authenticated customer download verification;
  • source-hidden .deb installation on clean Ubuntu 24.04 amd64 as a normal user;
  • hosted enrollment + fleet: one-time tenant-scoped codes (digest-only, revocable, atomically consumed), signed heartbeats with online/stale/offline/degraded/ never_seen/revoked states, node rename, and strict tenant isolation;
  • tenant-isolated meshes: a peer message is authorized by explicit mesh membership, not mere key trust or reachability; cross-tenant traffic is rejected before MissionEngine ingress unless an explicit shared mesh exists; revocation takes effect immediately; standalone meshes work fully offline;
  • the complete OTA MAC / link protocol (deterministic, no radiation): RTS/CTS reservation, carrier sense, randomized backoff, hidden-node mitigation, three-node contention with fairness/no-starvation, two-node TDD, BLOCK_ACK-bitmap selective retransmission, and the frame set BEACON/RTS/CTS/DATA/BLOCK_ACK/NACK/CANCEL/FIN/DELIVERY_RECEIPT;
  • the four acknowledgement layers — link ACK, delivery receipt, mission acknowledgement, mission result — kept distinct (a radio frame ACK is never reported as mission success);
  • operator-controlled physical TX (unchanged from beta.2): available once the local operator enables it with a TX-capable device, gated only by technical-correctness controls, with no vendor/cloud authorization;
  • RF research records (mesh-authorization, channel-access, RTS/CTS outcomes, collision recovery, retransmission, delivery latency, fleet health, cross-tenant rejection) with pseudonymized tenant/mesh/node ids and secret-scanning, and leakage-free mission-grouped dataset snapshots;
  • documentation command-regression testing against the packaged CLI.
Physical SDR transmission is available under local operator control. The operator enables it, selects all physical parameters, and is responsible for operating the radio per the rules at the operating location; Aithernet never chooses a frequency. Automated qualification uses hardware mocks + GNU Radio loopback and does not radiate; cabled, shielded and authorized-OTA tests are run by the operator on their hardware after installation. The optional live external-coordinator → coding-agent → generated-modem test uses the operator's own coordinator credential.

Genuine limitations that are not claimed:

  • invitation-only early access; physical TX is operator-controlled and OFF by default — the operator enables and configures it locally on TX-capable hardware;
  • no physical over-the-air transmission was performed during release qualification (automated tests use mocks + loopback and do not radiate); cabled/shielded/OTA testing is operator-run;
  • WSL is software-only and its audio endpoints are not SDR hardware;
  • physical Pluto / USB-SDR transmission was not tested on hardware here (operator-run);
  • real Gemini / Codex external-provider missions remain optional live canaries rather than mandatory deterministic release qualification;
  • full reboot / disconnect-reconnect, multi-node over-the-air, and long-duration soak validation are not claimed;
  • owner research collection is opt-in and owner-operated — not enabled for external customers by default;
  • Aithernet does not claim access to any model's hidden chain-of-thought; it records only provider-exposed reasoning and structured decision explanations, and never collects or uploads secrets.

Detailed, dated release notes for each early-access build are provided in the portal. This public page intentionally omits version-specific download details.
