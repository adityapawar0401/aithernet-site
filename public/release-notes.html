Release notes

What's new in Aithernet.

Releases are delivered to approved customers through the authenticated portal, where you'll also find the verification key and exact install commands for your release.

Current release

1.0.0-beta.4

Early access · current

Highlights

  • Provider-neutral agent runtime. Configure the coordinator and coding agent independently and mix providers freely — Claude, Codex, Anthropic, OpenAI, Gemini, any OpenAI-compatible endpoint, and local models.
  • Bring your own account. Use a Claude or ChatGPT subscription, a provider API key, a cloud account (Bedrock / Vertex), or your own local compute. No single vendor is required.
  • Clear capacity and fallback. See how each role's allowance is used, group roles that share an account into one capacity pool, and set explicit fallback chains.

New capabilities

  • First-class coordinator providers: Claude (Pro/Max), Anthropic API, Bedrock, Vertex, OpenAI API, OpenAI-compatible, local, Gemini API, Gemini Vertex, and the installed Google CLI.
  • First-class coding providers: Codex (ChatGPT), Codex/OpenAI API, Claude Code, and local.
  • Recommended presets and explicit per-role fallback chains — fallback is always recorded and never silent.
  • Accurate, per-provider billing labels (subscription allowance, separate API billing, cloud-provider billing, or local compute) shown during setup.
  • New agents commands: list-providers, inspect-provider, capacity, fallbacks, set-fallbacks, presets, disconnect.

Improvements

  • A single canonical, role-aware provider interface; coding capability is never inferred from the ability to produce text.
  • Clearer, normalized provider errors with concise operator actions — never exposing credentials.

Compatibility

  • Ubuntu 24.04 LTS (amd64).
  • Earlier releases remain available and unchanged.

Known limitations

  • Early access is invitation-only.
  • Credentials are referenced by name and stored only in the protected secret store — never in config, datasets, logs, artifacts, or the portal.
  • Cloud (Bedrock / Vertex) adapters are configurable now; live qualification of those exact methods is operator-run on your own cloud account.

Upgrade

Download the new package from the portal, verify it, and install it over your existing node. Setup, enrollment, and provider configuration are preserved.

sudo apt install ./aithernet_1.0.0~beta.4_amd64.deb

Previous releases

1.0.0-beta.3, 1.0.0-beta.2, 1.0.0-beta.1, and the 0.8.0 series remain available to approved customers in the portal. Detailed, dated notes for each release are provided there.

Engineering details — manifests, checksums, and verification records — are available with each release in the portal.
