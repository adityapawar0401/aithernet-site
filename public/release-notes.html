Release notes

What's new in Aithernet.

Releases are delivered to approved customers through the authenticated portal, where you'll also find the verification key and exact install commands for your release.

Current release

1.0.0-beta.3

Early access · current

Highlights

  • Account-bound fleets. Enroll nodes to your account and see them appear automatically, with clear online, stale, offline, and degraded status.
  • Tenant-isolated meshes. Organize nodes into meshes that explicitly authorize communication. Nodes outside a shared mesh cannot talk to each other.
  • Complete over-the-air link protocol. Robust SDR communication with channel reservation, acknowledgement, retransmission, and delivery semantics.

New capabilities

  • First-class mesh management in the portal and CLI — create meshes, add and remove nodes, and manage membership.
  • Node management: rename nodes, manage enrollment codes, and review mesh membership from the portal.
  • Fleet status with online, stale, offline, degraded, never-seen, and revoked states.
  • Standalone meshes that work fully offline, with no hosted account required.
  • Structured operational records and datasets for mesh, fleet, and link-layer activity.

Improvements

  • Clearer separation between message delivery and mission outcomes across the product.
  • Simpler, more consistent node and mesh presentation in the portal.

Compatibility

  • Ubuntu 24.04 LTS (amd64).
  • Earlier releases remain available and unchanged.

Known limitations

  • Early access is invitation-only.
  • Physical transmission is operator-controlled and configured locally on supported hardware.

Upgrade

Download the new package from the portal, verify it, and install it over your existing node. Setup and enrollment are preserved.

sudo apt install ./aithernet_1.0.0~beta.3_amd64.deb

Previous releases

1.0.0-beta.2, 1.0.0-beta.1, and the 0.8.0 series remain available to approved customers in the portal. Detailed, dated notes for each release are provided there.

Engineering details — manifests, checksums, and verification records — are available with each release in the portal.
