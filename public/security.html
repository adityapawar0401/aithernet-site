Security

Trust is explicit, not assumed.

Every node has a cryptographic identity, every message is authenticated, and communication is authorized by explicit mesh membership. Operators keep local control at all times.

Cryptographic node identities

Each node holds its own keypair and authenticates by signing. There is no shared secret to distribute or leak.

Explicit mesh membership

A mesh is the trust group that authorizes communication. Reachability alone grants nothing.

Authenticated peer messages

Every message is signed and bound to its sender, recipient, and mesh before it is accepted.

Tenant isolation

Nodes in different tenants cannot communicate unless an explicit shared relationship exists.

Replay protection

Messages carry nonces and sequencing, so captured traffic cannot be replayed.

Local operator control

Hardware and operating parameters are chosen locally. Nothing transmits without explicit local configuration.

Bounded execution

Agent and tool execution runs within configured limits and approvals.

Signed releases & artifact integrity

Packages are signed and verifiable; outputs are captured as integrity-checked artifacts.

Careful secret handling

Credentials stay local to the operator and are never embedded in packages or transmitted over the peer protocol.
