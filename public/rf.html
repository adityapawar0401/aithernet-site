Radio

Software and physical SDR, under operator control.

Aithernet supports simulated and physical SDR transports, with local operator control over hardware and operating parameters. The same authenticated peer protocol runs on a network or over the air.

Simulated and physical transports

Develop and validate against a software SDR transport, then move to physical hardware when you're ready.

Operator-selected hardware

You choose the device and the operating parameters. Physical transmission is enabled and configured locally on supported hardware.

Reference modem profiles

Start from reference waveform profiles, or develop your own protocols through the GNU Radio runtime.

GNU Radio integration

Signal-processing flowgraphs run inside a managed, observable runtime.

Mesh-authenticated communication

Over-the-air messages are authorized by the same explicit mesh membership used everywhere else.

Reliable delivery

The link layer provides channel reservation (RTS/CTS), acknowledgement, retry, and delivery semantics for robust exchange.
